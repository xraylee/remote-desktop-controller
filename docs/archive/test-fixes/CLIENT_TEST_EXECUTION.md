## 🔧 故障处理流程

### 问题 1: 应用启动失败

**现象**: Flutter 应用崩溃或无法启动

**诊断步骤**:
```bash
# 1. 检查 FFI 库
ls -lh ../../target/debug/librdcs_core.dylib

# 2. 查看详细错误
flutter run -d macos -v

# 3. 清理并重新运行
flutter clean
flutter pub get
flutter run -d macos
```

---

### 问题 2: 设备代码显示占位符

**现象**: 主页显示 `--- --- ---`

**诊断步骤**:
```bash
# 1. 检查终端日志，查找配置初始化日志
# 应该看到: ✅ Configuration initialized. Device code: XXX-XXX-XXX

# 2. 检查配置文件
cat ~/.rdcs/config.json | grep deviceCode

# 3. 如果 deviceCode 为空，删除配置重试
rm -rf ~/.rdcs/
# 重新启动应用
```

---

### 问题 3: 邀请码生成失败

**现象**: 点击按钮后显示错误或无响应

**诊断步骤**:
```bash
# 1. 查看终端是否有错误日志
# ❌ rdcs_generate_invite: null handle
# ❌ rdcs_generate_invite: engine is shutdown

# 2. 检查 FFI 实现
grep -A 10 "pub extern \"C\" fn rdcs_generate_invite" ../../crates/rdcs-ffi/src/lib.rs

# 3. 重新编译 FFI 库
cd ../..
cargo build -p rdcs-ffi

# 4. 重启 Flutter 应用
```

---

### 问题 4: 复制功能无效

**现象**: 点击复制按钮后无法粘贴

**诊断步骤**:
1. 检查是否有 SnackBar 提示
2. 尝试复制其他内容验证系统剪贴板是否正常
3. 查看终端是否有权限错误
