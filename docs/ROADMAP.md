# RDCS Roadmap

This document outlines the planned milestones for the **RDCS (Remote Desktop Control System)** project. The roadmap is a living document and may be adjusted based on community feedback, technical discoveries, and shifting priorities.

## Phase 1: Research & Planning (Q2 2026) - DONE

- [x] Market analysis and competitive landscape review
- [x] Product brainstorming and feature ideation
- [x] Initial project repository setup
- [x] Community feedback collection

## Phase 2: Design & Architecture (Q3 2026) - IN PROGRESS

- [ ] Requirements specification and product requirements document
- [ ] Technical architecture design
- [ ] UI/UX design and prototyping
- [ ] Protocol and relay server architecture
- [ ] Security model and threat analysis
- [ ] Development toolchain and CI/CD pipeline setup

## Phase 3: MVP Development (Q3-Q4 2026) - PLANNED

- [ ] Core relay server implementation
- [ ] Desktop client (Windows, macOS, Linux)
- [ ] Web-based admin dashboard
- [ ] Session management and authentication
- [ ] Basic remote desktop functionality (screen sharing, input forwarding)
- [ ] Docker deployment support
- [ ] End-to-end test suite

## Phase 4: Beta & Polish (Q1 2027) - PLANNED

- [ ] Closed beta testing with early adopters
- [ ] Performance optimization and latency reduction
- [ ] Comprehensive documentation (user guides, API reference, deployment guides)
- [ ] Security audit and penetration testing
- [ ] Bug fixes and stability improvements based on beta feedback
- [ ] Internationalization (i18n) support

## Phase 5: v1.0 Release (Q2 2027) - PLANNED

- [ ] Stable v1.0 release
- [ ] Mobile clients (iOS, Android)
- [ ] Ecosystem integrations (Slack, Microsoft Teams, LDAP/AD)
- [ ] Plugin and extension system
- [ ] Public documentation site launch
- [ ] Community contributor onboarding program

---

## Contributing to the Roadmap

The RDCS roadmap is shaped by community input. If you have ideas for features, integrations, or improvements that are not listed above, we encourage you to get involved:

- **Open a [Feature Request](https://github.com/your-org/rdcs/issues/new?template=feature_request.md)** to propose new items for the roadmap.
- **Join a [Discussion](https://github.com/your-org/rdcs/discussions)** to share your perspective on priorities and trade-offs.
- **Submit a pull request** to this document if you would like to propose changes directly (these will be reviewed by the maintainers).

We review and update the roadmap at the start of each quarter. Community suggestions that align with the project vision are welcome at any time.
