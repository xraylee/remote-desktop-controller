**版本**：v1.0

**状态**：草稿

## 1. 背景与目标

### 1.1 背景

小微企业（<50 人）在远程办公和 IT 支持场景中面临三重困境：商业远程桌面软件（TeamViewer $49/月、向日葵限速版）要么价格过高要么体验阉割；开源方案（RustDesk）需要自行部署服务器和维护，技术门槛超出非专业 IT 人员能力；最终大量团队退化为 QQ 远程 + 微信视频口述指导的低效模式。

横向对比 9 款主流产品后发现，市场上不存在一款同时满足"零部署、零注册、零安装、免费可用"的远程桌面产品。这是 RDCS 的核心机会。

### 1.2 目标用户

| 用户角色 | 特征描述 | 核心诉求 | 使用场景 |
|---------|---------|---------|---------|
| **工作室负责人** | 30 人以下团队老板，兼任 IT 管理，技术能力有限 | 免费或极低成本、不需要管服务器、能知道谁连了谁的电脑 | 远程帮员工修电脑、查看设计稿、指导新人操作 |
| **远程员工** | 设计/开发/运营岗位，不懂技术，只需要"被连接" | 不装软件、不注册账号、打开就能被远程 | 在家办公时请同事帮忙看电脑问题 |
| **临时协助者** | 老板、合伙人、外包技术支持 | 手机也能临时操作远程电脑、不用装 App | 在外面用手机帮员工解决紧急问题 |

### 1.3 业务目标与成功指标

| 目标 | 衡量指标 | 目标值 | 监测方式 |
|------|---------|--------|---------|
| 极速首次体验 | 下载到首次成功连接耗时 | < 2 分钟 | 客户端埋点 |
| 高连接成功率 | 跨网络连接成功率 | > 95% | 服务端日志 |
| 低成本获取 | 用户获取成本（CAC） | < 50 元/用户 | 渠道追踪 |
| 自然增长 | 用户推荐率（NPS） | > 50 | 季度调研 |
| MVP 交付 | 核心功能上线时间 | 6 个月内 | 项目看板 |

### 3.2 连接策略

| 层级 | 场景 | 连接方式 | 延迟预期 | 用户感知 |
|------|------|---------|---------|---------|
| **L1** | 同一局域网 | mDNS 自动发现 + TCP/UDP 直连 | < 5ms | "秒连" |
| **L2** | 跨网络，非对称 NAT | STUN/ICE UDP 打洞 | < 30ms | "正常连接" |
| **L3** | 跨网络，对称 NAT 或穿透失败 | 官方中继节点转发 | < 100ms | "正常连接" |

客户端在 500ms 内自动选择最优路径，用户侧只显示"已连接"，不感知底层路径。

### 3.3 权限模型

> **设计原则**：客户端无角色区分，所有用户看到相同的极简界面。管理功能统一在 Web 控制台完成。

| 功能 | 客户端（所有人） | Web 控制台（负责人） |
|------|----------------|-------------------|
| 发起/接受远程连接 | ✅ | — |
| 文件传输、剪贴板同步 | ✅ | — |
| 附近设备发现 | ✅ | — |
| 邀请码生成 | ✅ | — |
| 设备管理（添加/移除） | — | ✅ |
| 团队成员与权限配置 | — | ✅ |
| 连接记录与审计 | — | ✅ |
| 会话录制管理 | — | ✅ |

> **权限配置作用范围**：权限配置通过 Web 控制台设置，影响会话级别的功能开关（如某成员是否可使用文件传输、会话录制）。所有用户在客户端看到的界面完全一致，但部分功能按钮可能根据权限置灰。

### 3.4 免费版与付费版策略

**核心原则**：免费版在 P2P 直连场景下不限速、不限画质（1080P/60FPS）。仅在中转（L3）场景下对免费用户限速，因为中转带宽是运营成本。

| 功能 | 免费版 | 付费版 |
|------|--------|--------|
| P2P 直连画质 | 1080P / 60FPS（不限速） | 4K / 120FPS |
| 中转画质 | 720P / 30FPS | 不限速 |
| 并发连接数 | 5 个 | 不限 |
| 会话录制 | ❌ | ✅（AES-256 加密存储） |
| 会话录制回放 | ❌ | ✅ |
| 邀请码 | ✅ | ✅ |
| 文字聊天 | ✅ | ✅ |
| 文件传输 | ✅ | ✅ |
| Web 控制台 | 基础（设备列表、连接记录） | 完整（成员管理、权限配置、审计日志） |
| 连接记录保留 | 30 天 | 365 天 |
| TOTP 双因素认证 | ❌ | ✅ |
| 管理员操作审计 | ❌ | ✅ |

**定价参考**（待定，需市场调研验证）：
- 免费版：5 并发，P2P 不限速，中转限速
- 基础版：10 并发，2,980 元/年
- 专业版：20 并发，5,980 元/年
- 无限版：不限并发，12,800 元/年

### 3.5 自部署规划

MVP 和 V1.1 阶段采用**官方运营**模式——中转节点和信令服务由产品方统一运维，用户零部署。

**V2.0+ 规划**：为有数据主权需求的企业客户提供**可选的自部署方案**：
- Docker Compose 一键部署（信令 + 中转 + Web 控制台）
- 所有数据存储在客户自己的服务器
- 与官方运营版本功能一致
- 自部署客户可选择关闭遥测数据采集

## 4. 整体交互链路

### 4.1 全局用户旅程

```
下载/获取客户端
      │
      ▼
  双击运行（免安装）
      │
      ▼
  自动生成 9 位设备码
      │
      ├──→ 被人连接 ──→ 收到连接请求弹窗 ──→ 确认/拒绝
      │
      ├──→ 主动连接 ──→ 输入设备码 / 选附近设备 ──→ 等待对方确认
      │
      └──→ 管理操作 ──→ 打开浏览器进入 Web 控制台 ──→ 设备/成员/记录管理
```

### 4.2 子功能项详细流程

> **设计原则：设备码与邀请码的明确区分**
>
> 为避免用户混淆，两者在长度、格式、用途和视觉上均做了严格区分：
>
> | 维度 | 设备码 | 邀请码 |
> |------|--------|---------|
> | 长度 | 9 位数字（3 组，空格分隔） | 4 位数字（无分隔） |
> | 格式示例 | `123 456 789` | `5823` |
> | 生命周期 | 永久，安装时生成，不改变 | 10 分钟有效，一次性使用 |
> | 用途 | "我是谁"——标识本机身份 | "这次让你连"——临时授权他人 |
> | UI 呈现 | 主界面大号字体常驻展示 | 弹窗中显示，倒计时消失 |
> | 类比 | 类似手机号 | 类似短信验证码 |

#### 4.2.1 首次启动

| 步骤 | 操作 | 系统响应 | 异常处理 |
|------|------|---------|---------|
| 1 | 双击客户端程序 | 显示欢迎界面 + 申请系统权限 | 权限拒绝 → 提示必须授权才能使用 |
| 2 | 授权完成 | 自动生成 9 位设备码（格式 123 456 789） | 生成失败 → 显示"重试"按钮 |
| 3 | 自动注册到信令服务 | 设备码绑定到团队（如有邀请码）或独立状态 | 网络不通 → 显示"离线模式，仅支持局域网连接" |
| 4 | 进入主界面 | 显示设备码 + "等待连接"状态 | — |
| 5 | 首次启动引导 | 底部提示"管理你的团队 → 打开控制台"（仅首次显示） | 用户忽略 → 不影响使用，控制台入口始终在底部可访问 |

#### 4.2.2 发起远程连接

| 步骤 | 操作 | 系统响应 | 异常处理 |
|------|------|---------|---------|
| 1 | 点击"连接"按钮 | 弹出连接面板 | — |
| 2a | 输入对方 9 位设备码 | 验证设备码有效性 | 无效码 → "设备码不存在或已离线" |
| 2b | 或：从"附近设备"列表选择 | 自动填充设备码 | 列表为空 → "未发现附近设备，请确认在同一网络" |
| 3 | 发送连接请求 | 对方屏幕弹出"XXX 请求连接"提示 | 超时 30 秒 → "对方未响应，请稍后重试" |
| 4 | 对方确认 | 建立连接（自动选择 L1/L2/L3 路径） | 连接失败 → "网络连接失败，请检查网络" + 自动重试 |
| 5 | 连接成功 | 显示远程桌面画面 | — |

> **V1.1 扩展**：新增扫描二维码连接方式，配合微信小程序实现手机扫码直连。

##### 连接请求确认弹窗

被控端收到连接请求时，屏幕中央弹出确认弹窗：

```
┌─────────────────────────────────┐
│                                   │
│    有人请求连接你的电脑            │
│                                   │
│    张工 (设备码: 987 654 321)     │
│                                   │
│    [拒绝]          [允许连接]     │
│                                   │
│    30 秒后自动拒绝                │
└─────────────────────────────────┘
```

**交互规则**：
- 弹窗显示控制方名称（如已加入团队）或设备码
- "允许连接"按钮为主色高亮，"拒绝"为次要样式
- 30 秒无操作自动拒绝，控制方收到"对方未响应"
- 弹窗出现时播放提示音（可在设置中关闭）
- 多人同时请求连接时，排队展示，先处理第一个
- 当前已有连接进行中时，新请求显示"已有远程连接，是否接受新连接？"（接受后断开旧连接）

##### 被控端会话中体验

被控端（受控方）在连接确认后，回到正常桌面使用状态。

**屏幕捕获排除策略**：被控端的所有会话状态 UI（浮动条、通知 toast 等）不参与屏幕捕获流——控制端看到的画面中不包含这些 UI 元素，仅看到被控端的纯净桌面。这避免了双方 UI 叠加导致的视觉混乱和误触风险。

**浮动状态条**（连接建立时显示 3 秒后自动淡出）：
```
┌──────────────────────────────────────────┐
│ ● 张工 正在查看你的屏幕       [断开连接] │
└──────────────────────────────────────────┘
```
- 连接建立时顶部显示浮动条，3 秒后自动淡出消失
- 被控方需要时，可通过菜单栏/托盘图标重新唤出状态条
- 浮动条不参与屏幕捕获（控制端不可见）

**菜单栏/托盘常驻图标**（始终可见）：
```
macOS 菜单栏:  ...  [RDCS●蓝色]  WiFi  时间
                       │ 点击展开
                       ▼
              ┌────────────────────┐
              │ ● 张工 正在连接中   │
              │ 时长: 12:34        │
              │ ─────────────────  │
              │ [断开连接]         │
              └────────────────────┘
```
- 会话进行中，菜单栏/托盘图标变为蓝色脉冲状态
- 点击展开气泡：显示控制方名称、会话时长、断开按钮
- 这是被控方随时可查看和操作的持久入口

**远程光标标识**：控制方的鼠标在被控端屏幕上显示为带名称标签的光标（如"张工"），让被控方知道对方正在操作哪里。此标识不参与屏幕捕获。

**会话结束通知**：控制方断开后，被控端屏幕中央弹出 toast"张工 已断开连接，远程会话已结束，时长 XX 分钟"，3 秒后自动消失。此 toast 不参与屏幕捕获。

#### 4.2.3 邀请码

| 步骤 | 操作 | 系统响应 | 异常处理 |
|------|------|---------|---------|
| 1 | 点击"生成邀请码" | 生成 4 位数字口令，显示倒计时（10 分钟） | — |
| 1b | 点击"刷新"按钮 | 旧口令立即失效，生成新口令 | 频繁刷新 → 无限制，但每次刷新旧口令立即失效 |
| 2 | 复制/分享口令（微信/钉钉/短信） | 口令复制到剪贴板 | — |
| 3 | 对方输入口令 | 验证口令有效性 + 匹配设备 | 口令过期 → "口令已过期，请重新生成" |
| 4 | 验证通过 | 等同于设备码连接流程 | 口令已使用 → "该口令已被使用" |

#### 4.2.4 远程会话中操作

| 子功能 | 触发方式 | 操作说明 | 异常处理 |
|--------|---------|---------|---------|
| **鼠标控制** | 默认模式 | 移动/点击/拖拽远程鼠标 | 延迟过高 → 自动降低画质 |
| **键盘输入** | 默认模式 | 键盘输入直接传递到远程 | 输入法冲突 → 提供"远程输入法切换"按钮 |
| **文件传输** | 工具栏按钮 / 拖拽 | 拖拽文件到远程桌面自动传输 | 文件过大 → 显示传输进度条 + 取消按钮 |
| **剪贴板同步** | 自动 | 复制文本自动同步到对方剪贴板 | 大内容 → 提示"剪贴板内容较大，是否同步？" |
| **画质切换** | 工具栏按钮 | 手动选择"清晰优先"/"流畅优先"/"自动" | — |
| **全屏模式** | 双击 / F11 | 远程桌面占满本地屏幕 | ESC 退出全屏 |
| **多显示器** | 工具栏按钮 | 切换远程机器的不同显示器 | 对方单显示器 → 按钮置灰 |
| **会话录制** | 工具栏按钮 | 开始/停止录制当前会话 | 磁盘空间不足 → 提示并自动停止 |
| **文字聊天** | 工具栏按钮 | 打开侧边聊天面板 | — |
| **发送 Ctrl+Alt+Del** | 工具栏按钮 | 向远程发送系统热键 | 仅 Windows 被控端支持 |

> **工具栏溢出策略**：窗口宽度 < 600px 时，低频操作（录制、聊天、多显示器、Ctrl+Alt+Del）收入"更多 ▾"下拉菜单，仅保留高频操作（全屏、文件、剪贴板、画质、断开）。

#### 4.2.5 连接状态展示

**主界面底部**：三种连接路径以小标签形式展示，视觉权重低于主内容区域。当前使用的路径用主色高亮，其余路径灰色弱化。

| 连接路径 | 展示方式 | 视觉权重 |
|---------|---------|---------|
| **局域网直连（L1）** | 小标签，显示延迟 | 当前使用时主色高亮，否则灰色 |
| **P2P 穿透（L2）** | 小标签 | 灰色 |
| **官方中转（L3）** | 小标签 | 灰色 |

**设计原则**：连接状态是辅助信息，不是用户第一关注点。采用小字号（11px）、浅色背景、紧凑排列，不抢占主界面的视觉焦点。

**远程会话中**：底部状态栏显示当前连接方式和性能指标（见 4.2.6 性能指标动态反馈）。

**异常场景**：当连接建立失败或网络质量下降时，在主界面底部出现醒目提示文案。

#### 4.2.6 性能指标动态反馈

远程会话界面的底部状态栏实时显示性能指标，并根据阈值动态变化视觉样式：

| 指标 | 正常（绿色） | 警告（黄色） | 危险（红色） |
|------|------------|------------|------------|
| **延迟** | < 50ms | 50-150ms | > 150ms |
| **帧率** | > 30fps | 15-30fps | < 15fps |
| **连接方式** | L1/L2（绿色圆点） | L3（黄色圆点） | 重连中（红色闪烁） |

**异常提示规则**：
- 延迟 > 150ms 或帧率 < 15fps 时：顶部弹出醒目横幅"⚠️ 网络不稳定，已自动降低画质以保持流畅"
- 延迟 > 300ms 或帧率 < 5fps 时：横幅变为红色"⚠️ 网络连接质量较差，远程操作可能卡顿"
- 横幅在指标恢复正常后 3 秒自动消失
- 状态栏中异常指标的数值变为对应颜色（黄/红），正常指标保持白色

#### 4.2.7 连接断开与重连

| 场景 | 系统行为 | 用户提示 |
|------|---------|---------|
| 网络波动（<5 秒） | 自动降低画质 + 缓冲 | 顶部黄色提示条"网络不稳定，正在优化画质" |
| 网络断开（>5 秒） | 显示"连接已断开" + 自动重连倒计时 | 全屏遮罩 + "正在重连…3,2,1" |
| 重连成功 | 恢复远程桌面 | 提示条"已重新连接"，3 秒后消失 |
| 重连失败（3 次） | 停止重连 | "无法重新连接，请检查网络后手动重试" |

6. **安全睡眠**：被控 Mac 进入睡眠时，自动断开并通知控制方"对方电脑已进入睡眠"

1. **UAC 弹窗处理**：普通用户模式下无法远程操作 UAC 弹窗。如需此功能，需安装"服务模式"（可选，需管理员权限安装）。服务模式安装后，UAC 弹窗可在远程会话中正常显示和操作
2. **Ctrl+Alt+Del**：通过工具栏按钮发送。远程 Windows 收到后显示安全选项界面（锁定/切换用户/任务管理器）
3. **多显示器**：工具栏提供显示器切换下拉菜单，显示远程机器的所有显示器编号和名称。支持"查看所有显示器"（缩小拼合）和"查看单个显示器"两种模式
4. **高 DPI 适配**：远程画面在 Windows 高 DPI 设置下正确缩放，不模糊
5. **安全模式重启**：付费功能——可向远程 Windows 发送"安全模式重启"指令，重启后自动重新建立连接

## 8. 技术选型约束与平台规划

### 8.1 技术选型约束（技术设计阶段评估）

以下技术选型方向需在技术设计阶段进行详细评估和选型，PRD 仅明确约束边界：

| 领域 | 候选方案 | 选型要求 |
|------|---------|---------|
| **客户端框架** | Flutter / React Native / 原生开发 | 需支持 macOS、Windows、Linux、iOS、Android 全平台；核心引擎跨平台复用，仅 UI 层和平台适配层按平台实现 |
| **服务端技术栈** | Node.js / Python / Rust / Golang 等 | 需支持高并发连接、低延迟转发、易于部署和维护 |
| **核心传输协议** | **WebRTC**（强制要求） | 远程桌面画面传输必须基于 WebRTC，确保低延迟和实时性 |

**选型原则**：
- 核心引擎（连接、编解码、加密、传输）必须跨平台复用，覆盖桌面和移动端
- 优先选择跨平台能力强、能统一桌面和移动端的技术方案
- 服务端需支持水平扩展，以应对未来用户增长
- WebRTC 作为画面传输的底层协议不可妥协，其他协议（如信令、文件传输）可灵活选择

## 13. 风险与依赖

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| P2P 穿透成功率低于 60% | 中 | 高（中转成本增加） | 提前部署多区域中继节点，优化 ICE 策略 |
| 高效编解码 CPU 占用过高 | 中 | 中（低配机器卡顿） | GPU 硬件加速 + 编码降级方案 |
| 中转带宽成本超出预算 | 高 | 高（商业模型不可持续） | ① 免费版中转限速 720P/30fps ② 付费版覆盖成本 ③ 高效编码降低 40-60% 带宽 ④ MVP 阶段建立单用户月均带宽成本模型，验证盈亏平衡点 |
| WebRTC 弱网表现不佳 | 中 | 中（丢包、抖动导致画面卡顿） | WebRTC SVC 自适应码率 + 前向纠错（FEC）+ NACK 重传 |
| 微信小程序审核不通过 | 中 | 中（延迟上线） | 同步开发钉钉/飞书版本 |
| 跨平台屏幕捕获兼容性 | 低 | 中（需平台适配） | 核心层抽象平台差异，各平台独立实现捕获层 |
| macOS 权限策略变化 | 低 | 高（功能受限） | 关注 WWDC 更新，预留权限适配方案；客户端检测权限状态，未授权时在界面显著位置持续提示 |
